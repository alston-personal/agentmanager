# 🏗️ Antigravity Agent OS: Data Layer (v1.1)

This repository serves as the **Data / Memory Root** for your Agent OS.

## 📂 Structure Overview
- `ideas/`: Raw concepts and brainstorming artifacts.
- `specs/`: Refined technical specifications and requirements.
- `projects/`: Active development repositories (as symlinks or subdirs).
- `validation/`: QA results, test reports, and verification evidence.
- `memory/`: Core agent memory (Identity, Context, Knowledge).
- `logs/`: Runtime logs and performance snapshots.

## 🧠 Memory Protocol
The agent uses the files in `/memory` to maintain state across sessions. Do not delete `session_sync.md` as it is the primary high-speed context buffer.

## 🚀 Lifecycle
Ideas can be promoted to Specs, and then to full Projects using the OS migration tools.
